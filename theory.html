DevTools

Что это

DevTools — набор инструментов веб‑разработчика, встроенных в браузер. Он помогает инспектировать DOM/CSS, выполнять JavaScript, анализировать сеть и хранилища, а также диагностировать проблемы производительности и поведения страницы.

Для QA DevTools полезен, чтобы быстро локализовать дефект, проверить запросы/ответы и понять, что реально хранится на клиенте.
Вкладка Elements

Инспектирование DOM и CSS

Вкладка Elements позволяет просматривать структуру страницы (DOM), менять атрибуты, редактировать HTML, включать/выключать CSS‑правила, смотреть computed‑стили и псевдо‑элементы (::before/::after).

Быстрые приёмы:

  • Поиск элементов (Ctrl/Cmd+F в Elements) по CSS‑селекторам.
  • Редактирование стилей прямо в панели Styles, включая состояния :hover/:active.
  • Проверка скрытия: display/visibility/opacity/aria-hidden и т.п.

Примеры CSS‑селекторов для поиска:

  • .class-name — поиск по имени класса
  • #id-name — поиск по id
  • div > div > h2 — поиск по иерархии
Вкладка Console

Выполнение JS и диагностика

Console — место для выполнения JavaScript в контексте текущей страницы. Тут удобно читать логи, проверять значения, вызывать отладочные функции приложения и быстро проверять гипотезы.

  • В тренажёре есть объект window.Trainer — его функции нужно вызывать в заданиях.
  • Полезно помнить про $0 (последний выбранный элемент в Elements) и copy(...) (копирование значения).
Вкладка Network

Анализ сетевых запросов

Network показывает все сетевые запросы страницы: URL, метод, статус, время, заголовки, параметры, payload и ответ. Это основной инструмент для анализа взаимодействия клиента и сервера (или, как в этом тренажёре, имитации API).

  • Request: URL, query params, request headers, payload.
  • Response: status code, response headers, body (Preview/Response).
  • Timing: где тратится время (DNS/SSL/TTFB/Download).
  • Preserve log помогает не терять запросы при навигации/перезагрузке.

Важно: в приложении запросы идут на /api/mission и обрабатываются Service Worker — поэтому это всё работает без бэкенда, но в Network всё равно видно.

Вкладка Application

Хранилища на стороне клиента

Application помогает работать с клиентскими хранилищами: Cookies, Local Storage, Session Storage, IndexedDB, Cache Storage и т.д. Эти хранилища решают разные задачи: хранение настроек, данных сессии, кэширования и оффлайн‑режимов.

Cookies

Cookies — небольшие фрагменты данных (обычно до ~4KB на cookie), которые браузер может автоматически отправлять вместе с HTTP‑запросами к домену (с учётом Path/Domain и политики SameSite).

  • Применение: сессии, персонализация, аналитика/трекинг.
  • Безопасность: флаг HttpOnly запрещает доступ из JS; Secure — только по HTTPS.
  • Срок жизни: до истечения expires/max-age или до закрытия браузера (session cookie).

localStorage

localStorage — синхронное key‑value хранилище в рамках origin (домен+протокол+порт). Данные живут без срока, пока их явно не удалить.

  • Применение: настройки, состояние UI.
  • Объём: обычно 5–10MB на origin (зависит от браузера).
  • Важно: не хранить секреты/токены, т.к. доступно из JS и уязвимо к XSS.

sessionStorage

sessionStorage похож на localStorage, но живёт только в рамках текущей вкладки/окна и очищается при закрытии вкладки.

  • Применение: временные данные шага/формы.
  • Объём: обычно до ~5MB на вкладку (зависит от браузера).

IndexedDB

IndexedDB — асинхронная NoSQL‑БД в браузере для структурированных данных. Подходит для больших объёмов, оффлайн‑кэша, сложных объектов и индексов.

  • Применение: оффлайн‑данные, кэш API, крупные списки/объекты.
  • Особенности: транзакции, object stores, индексы, асинхронный API.
  • В DevTools: Application → IndexedDB позволяет просматривать записи.
